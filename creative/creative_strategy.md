## Locked Operating Principles (2026-06-22)

1. **Two different structures, not one escalator.**
   - Tier 1 (zero → competent): solved, borrowable — courses, craft frameworks, existing curricula. Adopt, don't invent.
   - Tier 2 (competent → masterpiece): never existed for anyone, including the masters studied. Built only by walking it. Do not wait to find this before starting Tier 1.

2. **Capability is built by doing, not consuming.** Reading/researching a great analysis produces the *feeling* of understanding without the muscle. Only self-generated synthesis (writing the rules, finding the constraint, building the confrontation) builds the transferable skill.

3. **The exist-vs-become rule for AI use in this pillar:**
   - Goal = a thing exists / a real problem gets solved (tools, side projects) → use AI fully, output-focused.
   - Goal = becoming someone who can do the thing (this pillar, specifically) → the core craft must be self-generated. AI's role here is periphery and critique only: research seeds, options to react to, sparring partner that attacks weak points. AI never produces the core artifact.
   - The Masterpiece dream is explicitly a mastery dream — if AI makes it, the mastery isn't real, and the result will land exactly like TFT Master rank: hollow, because there is no self-authorship.

4. **The unit of work is the confrontation, not the system or the scene.**
   - A power system has no quality in isolation — "balanced" only means something relative to a specific conflict it has to resolve. System-first design failed before: infinite possibilities, every problem equal weight, no definition of done.
   - A raw emotional scene gave an anchor but no standard above the median — "lame" with no diagnosable cause.
   - A confrontation (two characters, constrained abilities, an apparent mismatch resolved through clever-and-fair exploitation of the constraints) supplies all three at once: a definition of done (someone wins or doesn't), a feedback signal (clever-and-fair vs. asspull), and it sits inside the existing elite-standard zone (strategic/systems judgment), not the median-standard zone (raw prose/emotion).

5. **Design the fight, let it force the system — not the reverse.** Don't invent a system and hunt for uses. Pick the confrontation first; build only the rule the confrontation demands.

6. **"Lame" is now a diagnostic, not a verdict.** When a draft reads as lame, the cause is locatable: an asspull resolution, constraints that don't actually collide, or an outcome that was obvious from the start. Debug the specific cause instead of treating the feeling as a stop sign.

7. **Watch for a new costume on the old paralysis pattern.** Reframing the approach (system → scene → confrontation) was legitimate triangulation this time because each reframe fixed a named failure of the last. If a fourth reframe is reached for before one confrontation is finished, treat the reaching-for-a-new-frame itself as the signal to stop and produce instead.

## The First Brick (Ready to Start)

**Project:** Design one confrontation that a power system has to make brilliant.

**Spec:**
- Two characters, each with one constrained ability.
- Character A should lose on paper (apparent power/resource mismatch).
- A wins by exploiting the *interaction* of the constraints — cleverly and fairly, not through a new rule introduced at the last second.
- Borrow-and-twist existing abilities for speed if needed — originality is not the point of brick #1.

**Output:** ~1 page — the two abilities, the apparent mismatch, the resolution, and why the resolution is fair (what rule made it possible, established before the payoff).

**Definition of done:** the resolution reads as clever *and* fair by the builder's own elite standard — not an asspull. If it reads as lame, name the specific cause (asspull / non-colliding constraints / obvious outcome) and revise that specific thing.

**After this brick:** let the next confrontation reveal what rule/system question needs solving next. Do not pre-build a roadmap beyond one brick at a time — direction emerges from the work, consistent with the bottom-up rule for exploration/unknown territory.
