## The Paralysis Pattern (Know This When Lost)

**The trap:** Needs architectural confidence before execution can start → full blueprint feels too big to complete → paralyzed → abandons.

**Why pure artisan-mode also fails:** Writing scenes without a structural anchor immediately triggers "what does this serve?" → kills builder's joy → stalls.

**The root:** Waterfall thinking in a domain that requires Agile.

**The fix: Minimum Viable Architecture (MVA)**
> Design *just enough* to give the architect brain permission to lay the first brick. Not all answers — just a compass heading.

MVA = four things only:
1. **Thematic core** — one sentence: "This story is about ___."
2. **Protagonist's want + flaw** — what they chase and what's broken in them
3. **Emotional endpoint** — how the reader should feel at the very end
4. **Core power system rule** — one foundational principle, not the full system

Then: write one piece → learn → update the architecture → repeat. **The architecture is a living document, not a locked blueprint.**

## Creation Process (Locked)

```
MVA (just enough architecture)
  → Generate one scene/artifact with AI
  → Curate against taste and vision
  → Update architecture from what was learned
  → Repeat
```

**Owner's role at every step:** Vision keeper + quality gate. Not a PM managing others from outside — the architect reviewing the work and fighting for the standard.

**AI's role:** Reasoning partner (answer architectural questions, resolve blockers) + execution partner (generate draft material). Same model as how this Life Strategies workspace was built.

## Governing Principle

**Art-First, Commerce-Second.** The creative work must never be driven by financial incentive. Career is the engine that funds and protects the time for this. If creative work naturally generates income one day — welcome. It must never dictate creative direction.

## Key Patterns to Watch

- **The Waterfall reflex:** Trying to answer chapter-200 questions before writing chapter 1. Counter: scope down to MVA and start anyway.
- **The abandonment loop:** Blueprint too big → paralyzed → leaves the work. Counter: AI-assisted reasoning session to reach "good enough" confidence, same as this workspace.
- **Money pressure creeping in:** Thinking "I need this to eventually make money." Counter: re-read the governing principle. Career is the engine. This must stay clean.
- **Artisan envy:** Feeling like "real creators" work differently (scene by scene, pure flow). Counter: the process must match the actual engine, not the romanticized version.
